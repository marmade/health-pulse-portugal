## Decisões tomadas

1. **Dados 100% reais** — zero mock data, zero Math.random, zero fallbacks silenciosos
2. **Arquivo server-side** — workflow gera tudo, não depende de visitas ao browser
3. **MSD Manuals como fonte prioritária** — peer-reviewed, português, gratuito
4. **Thresholds realistas** — 30/50/40% em vez de 150/250%
5. **Mural acumulativo** — keywords históricas nunca desaparecem
6. **Supabase Lovable** — mantemos por agora; migração para o Supabase da Marta quando for para produção
7. **Bookmarks com badges de eixo** — mapeamento frontend (sem colunas na DB do Lovable)
8. **Benchmark negativo com contra-narrativa** — link MSD em cada pseudociência
9. **Guiões automáticos** — workflow gera, comunicador revê
10. **/sobre editável no admin** — DB ganha sobre ficheiro, base limpa
