# Sessão 4 — 2026-04-12

## Objectivo
Migração completa do backend Supabase: da instância Lovable (`cyjwhmuakmiytypewwfw`) para a instância da Marta (`ijpxjpbjudaddfatibfl`).

## Contexto
O projecto usava duas instâncias Supabase separadas. A instância Lovable não aparecia no dashboard da Marta, dificultando a gestão. A instância da Marta já tinha 19 tabelas criadas mas com schemas diferentes do código React e dados parciais.

## Trabalho realizado

### 1. Mapeamento completo
- Inventariadas 19 tabelas, 38 migrations, 7 edge functions, 5 scripts Python
- Comparação coluna a coluna entre as duas instâncias
- Descoberta: **17 de 19 tabelas** tinham schemas incompatíveis (nomes de colunas diferentes, colunas em falta)
- Apenas `bookmarks` e `eixos_archive` tinham schema idêntico

### 2. Preparação de artefactos
- `scripts/migration_consolidada.sql` — SQL consolidado das 38 migrations (749 linhas, idempotente)
- `scripts/migrate_data.py` — script de exportação/importação com paginação e upsert
- `scripts/switch_supabase.sh` — script de troca de configuração com backups

### 3. Exportação e backup (dados da Marta)
- Exportados 5 tabelas com dados: keywords (83), debunking (36), guioes (100), revisao_pares (4), textos (2)
- Guardados em `scripts/marta_backup/*.json`
- Verificação de integridade antes de prosseguir

### 4. Drop + recreate (17 tabelas)
- Drop CASCADE de todas as tabelas excepto bookmarks e eixos_archive
- Recreação com schema Lovable (tabelas, FKs, indexes, RLS, 63 policies)
- Executado via Supabase Management API (`execute_sql`)

### 5. Transformação e reimportação (dados da Marta)
- **debunking** (36 rows): mapeamento `titulo→term/title`, `fonte→source`, `url_fonte→url`, default `classification='a verificar'`
- **guioes** (100 rows): preservado `referencia_url` dentro de `referencia_cientifica` (campo combinado)
- Ambos inseridos com sucesso na nova estrutura

## Regra de merge aplicada
- Em caso de conflito por ID, ganha a instância com mais registos nessa tabela
- debunking e guiões → Marta ganha (transformados para schema Lovable)
- bookmarks → Lovable ganha (upsert merge)
- Todas as outras → Lovable (única com dados)
